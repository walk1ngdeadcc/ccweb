# CAN 协议基础

## 一句话总结

CAN 是一种多主异步串行通信总线，通过非破坏性仲裁实现多节点同时发送而不冲突。

## 数据帧格式

```
起始位 | 仲裁段(11/29bit ID) | 控制段 | 数据段(0-8字节) | CRC | ACK | 结束位
```

## 核心特性

1. **多主通信**：任何节点都可以主动发送
2. **非破坏性仲裁**：ID 越小优先级越高，冲突时高优先级继续发送
3. **错误检测**：CRC、位填充、ACK、位监控、格式检查
4. **自动重发**：错误帧发送后自动重传

## 面试回答

CAN 总线采用差分信号传输，抗干扰能力强。仲裁机制是核心：当多个节点同时发送时，ID 小的节点会覆盖 ID 大的节点，高优先级节点不会被打断。这种非破坏性仲裁保证了实时性，是 CAN 适合汽车电子的关键原因。

## 与 UART/SPI/IIC 对比

| 特性 | CAN | UART | SPI | IIC |
|-----|-----|------|-----|-----|
| 线数 | 2 | 2 | 4 | 2 |
| 多主 | 是 | 否 | 否 | 是 |
| 速率 | 1Mbps | 常见 115200 | 可达 50MHz | 3.4Mbps |
| 距离 | 可达 1km | 15m | 短 | 短 |
| 仲裁 | 硬件仲裁 | 无 | 无 | 软件仲裁 |
